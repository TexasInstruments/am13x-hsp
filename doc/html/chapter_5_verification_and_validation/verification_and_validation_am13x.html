Debug and Customize Simulink Generated Code in CCS

After building your model from matlab, use the following steps to import and debug the generated code
Step 1: Go to file -> import projects...
Step 2: Click on Browse, and navigate to the CCS_Project folder under the {example_name}_ert_rtw project folder. Select the Folder
Step 3: You will be able to see the projectspec file being picked up by the ccs. Click on the file and then click on Finish.
Now you should see the project being imported in ccs. You can now 'right click' on the project to debug the project in CCS.

External Mode

Monitor and Tune (External Mode) simulation enables you to tune model parameters and evaluate the effects of different parameter values on the model results in real-time. This external mode establishes a communication channel between Simulink on your development computer (host) and the target hardware that runs the executable file created by the code generation and build process in real time.
Monitor and Tune accelerates parameter tuning. You do not have to rerun the model each time you change parameters. You can also use Monitor and Tune to develop and validate your model using the actual data and hardware for which it is designed. This software-hardware interaction is not available solely by simulating a model.
