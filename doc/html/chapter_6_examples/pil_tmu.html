PIL TMU

Model:

Description:

This example demonstrates Processor-in-the-Loop (PIL) code verification with TMU (Trigonometric Math Unit) code replacement on AM13E230x hardware. PIL runs the generated code on the target hardware and compares results with Simulink simulation to verify numerical equivalence and measure execution time using hardware profiling.
Running Steps
1. Hardware Setup
  • Connect AM13E230x board to PC via USB
  • Note the COM port number from Device Manager → Ports (COM & LPT)
2. Configure PIL Settings
  • Open the model and navigate to: Hardware Settings → Target Hardware Resources → Connectivity
  • Set COM port and baud rate
  • Or modify in MATLAB:codertarget.data.setParameterValue(bdroot, 'Connection.COMport', 'COMx')codertarget.data.setParameterValue(bdroot, 'Connection.Baud', 'xxxxxx')
More information about connectivity, click Here
Note: if PIL connection fails, resetting the board and trying again might fix the issue
3. Configure SysConfig
  • Ensure the syscfg file is linked: Hardware Settings → Target Hardware Resources → SysConfig → SysConfig Project File
  • Click Launch to verify GPIO and peripheral configurations
4. Check for the TMU Code Replacement
  • Navigate to: Configuration Parameters → Code Generation → Interface → Code Replacement Library
  • Check if the TMU optimization library (AM13x_TMU) for AM13E230x is selected
5. Run PIL Simulation
  • Open Apps → SIL/PIL Manager
  • Select SIL/PIL Mode → Processor-in-the-loop (PIL)
  • Configure:
  • System Under Test: Select model blocks
  • Stop Time: Top model
  • Click Run Verification
6. View Results
  • Check Simulation Inspector for PIL vs. normal mode comparison
  • Review Code Generation Report for TMU code replacements
  • Analyze execution time profiling data
