LED Toggle 1 Hz

Model:

Description:

The LED Toggle 1 Hz toggles 2 leds at 1 Hz and 2 Hz respectively.
It uses scheduler at 1 Hz, and initializes the GPIO with the syscfg file provided with the example. The example is already linked to the syscfg file, which can be seen at Hardware Settings -> Hardware Implementation -> Target Hardware Resources -> SysConfig -> SysConfig Project File.
You can also click on launch, to read or change the initialization.
In simulink, 2 pulse generators are available with sample time 1 and 2, which go to C function block. In the block under Code Generation, We check the value of pulse and set or clear the LED pin based on the value. Under C Function -> Code Generation -> Code Generation Custom Code -> Headers, we have also added headers for the example to work. Both C functions do the same task with 2 different LEDs.
In this example, Syscfg file is used to initialize code. It initializes GPIO and sets the MCLK_FREQ_HZ to 200MHz
Pulse Generators are used to configure the timer. The lowest sample time (1s in the example) is multiplied with MCLK_FREQ_HZ set by the syscfg.
This is the number of ticks that is passed to the SYSTICK timer. The conversion from seconds to ticks is automatically handled by the am13x_tick.c file.
User only need to provide sample times needed, e.g. 1s, 0.001s, 10s etc.
SYSTICK calls SysTick_Handler which then calls the rt_OneStep() function.
rt_OneStep() is an important function, as all the runtime code e.g. code written in custom code, will be inside this function.

Example Path:

slx file: examples/led_toggle_1Hz/led_toggle_1Hz_am13x.slx
syscfg file: examples/led_toggle_1Hz/led_toggle_1Hz_am13x.syscfg
