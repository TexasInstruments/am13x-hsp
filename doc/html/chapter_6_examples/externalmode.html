External Mode

Model:

Description:

This example demonstrates External Mode code for monitoring and tuning variables on AM13E230x hardware. The generated code runs on the target hardware and logged signals can be monitored on the data inspector and measure run time execution.
Running Steps
1. Hardware Setup
  • Connect AM13E230x board to PC via USB
  • Note the COM port number from Device Manager → Ports (COM & LPT)
2. Configure Connectivity Settings
  • Open the model and navigate to: Hardware Settings → Target Hardware Resources → Connectivity
  • Set COM port and baud rate
  • Or modify in MATLAB:codertarget.data.setParameterValue(bdroot, 'Connection.COMport', 'COMx')codertarget.data.setParameterValue(bdroot, 'Connection.Baud', 'xxxxxx')
More information about connectivity Here
5. Run External Mode:
  • Click Monitor & Tune button from HARDWARE Toolstrip to run External Mode
6. View Results
  • Check Simulink Data Inspector Inspector
  • Analyze execution time profiling datas

Limitation:

  • XCP adds overhead and increase CPU load which causes data drops over serial. Reduce sample time to avoid data loss or connection issues.
  • "Measure task execution time" is possible, but enabling "Measure function execution times" affects External Mode connection. Use this settings only during PIL
Monitor and Tune accelerates parameter tuning. You do not have to rerun the model each time you change parameters. You can also use Monitor and Tune to develop and validate your model using the actual data and hardware for which it is designed. This software-hardware interaction is not available solely by simulating a model.
